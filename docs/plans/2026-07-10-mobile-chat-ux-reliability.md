# 2026-07-10 微信式移动聊天与消息可靠性修复计划

## 关联需求
- `UX-001`：移动优先的微信式会话列表、聊天页与底部导航。
- `UX-005`：聊天页仅保留当前联系人高频动作。
- `UX-006`：设置在移动端使用全屏页面。
- `UX-007`：所有用户可见文案进入四语 i18n。
- `FR-MSG-001`：会话行显示真实未读、时间、置顶和最后消息。
- `FR-MSG-004`：出站消息状态稳定显示。
- `FR-MSG-006`：出站乐观消息与服务端消息幂等合并。
- `FR-MSG-010`：发送成功返回并使用真实本地/平台消息 ID。
- `SEC-001`：前端不得展示服务器敏感路径和完整管理命令。

## 实施步骤
1. 先新增回归测试，覆盖移动列表/聊天两级导航、TabBar 隐藏、临时消息禁止翻译、发送 ID 替换、置顶去重、已发送状态保留、五分钟时间分组、四语键和移动全屏设置。
2. 修复 P0：移动端回列表路径；临时/失败/发送中消息不进入翻译；真实 ID 替换乐观 ID。
3. 修复 P1：置顶只渲染一次；未读改红点；发送状态合并；时间和翻译去噪；清理硬编码文案。
4. 修复 P2：textarea 使用 scrollHeight；三模式并排；手机头部动作收敛；设置全屏；敏感路径进入高级折叠。
5. 执行 Web 单测、i18n 一致性、Python 回归和生产构建。
6. 用 390×844 和 1440×900 浏览器执行主链路验收，检查网络中无 `tmp-* /translate` 请求。
7. 部署、健康/静态资源验证，更新 SDD 和项目四文件，提交推送。

## 验收矩阵
- 手机 Chats 首屏显示会话列表；选中后只显示聊天并隐藏 TabBar；返回恢复列表和 TabBar。
- 桌面保持双栏。
- 临时、pending、failed 消息永不调用翻译 API。
- 发送成功后乐观气泡使用后端真实 `local_message_id/message_id`。
- 置顶项只出现一次，时间/最后消息/未读红点完整。
- 服务端刷新后最后一条本人消息继续显示已发送。
- 气泡时间为 `HH:MM`，同发送方五分钟内仅最后一条显示。
- 相同译文不显示；隐藏翻译仅 hover/激活时显示。
- 泰语模式下指定区域无中英文硬编码。
- textarea 自动增高且上限 140px。
- `npm --prefix web run build` 通过。
